Steam Price Watcher — Changelog
===============================

v0.1.4.1 (2026-07-03)
- Fixed «vanishing» of newly-added purchases 15–30 s after startup:
  the background backfill thread was overwriting the files with its
  own stale snapshot, wiping out any operations the user did while
  it was running. It now merges instead of overwriting (purchases,
  watchlist and salelist).
- Fixed several finance-block bugs on the History tab:
  * «Steam balance» only appeared after a GUI restart when the
    anchor was configured;
  * avatar-widget balance stayed at 0.00 on an expired session
    instead of falling back to the anchor + operations since;
  * balance colour was compared against the anchor instead of zero
    (now: +green, −red, 0 neutral);
  * add/delete/edit of a record didn't update the avatar-widget
    balance until restart;
  * IndexError in the block's layout from recursive Configure events.
- The balance anchor now stores an exact `created_at` moment, and
  records are filtered by their own `_created` timestamp (the moment
  they were added) rather than the operation date. A back-dated entry
  logged AFTER the anchor still counts.
- The «Оновити» button on the Log tab now always jumps to the end
  (previously it kept the current scroll); auto-refresh of the Log
  is skipped when the file hasn't changed — reading mid-log no
  longer snaps back to the bottom.
- Cosmetic: rows with 🚫 «don't check» or 🔇 «don't notify» are now
  tinted icy silver («frozen»). The finance block goes 2×2 when an
  anchor is set with vertically-aligned «|» separators; total
  purchases coloured blue (Buy tab button colour), total sales
  yellow; the «Balance» cell is now «Steam balance»; the anchor
  footnote is left-aligned.

v0.1.4.0 (2026-07-01)
- History now supports manual «Add» — a new button on the first row (also
  in the context menu). Accepts Steam Market URLs (cards) or Steam Store
  URLs (games); metadata is pulled automatically.
- History buttons split into two rows: row 1 — «Purchase history / Export
  CSV / Add», row 2 — «Re-add to watchlist / Edit price / Delete».
- New «Balance» cell. A ⚙ button next to the finance block opens the
  «balance anchor» dialog: enter your total purchases, total sales and
  Steam wallet balance as of a given date — all subsequent operations
  are added to those numbers automatically. A footnote «* adjusted as
  of: DD.MM.YYYY» appears below the block. Reset via the same ⚙ button.
  «Spent» is computed automatically; without an anchor the finance block
  works exactly as before.

v0.1.3.3 (2026-06-29)
- Fixed daily-spam of "reminder" notifications for the same active game
  sale: previously the same alert arrived every day because the antispam
  reminder (24h by default) matched the games poll interval. For games
  and Epic-cheaper alerts, the reminder branch now skips when the price
  is unchanged — a notification arrives when a sale starts, when the
  price drops further (Steam sometimes deepens a discount), or when the
  sale ends and a new one begins. Cards (buy/sell) are unchanged —
  reminders there stay useful.

v0.1.3.2 (2026-06-20)
- Fixed: re-adding a card after buying/selling it — the program said
  "already in the list" because it counted closed (bought) history rows
  as active.
- "Copy link" added to the right-click menu on Purchases, Sales and
  History tabs.
- History column rename: "Card" → "Card/Game", "Game name" → "Name".

v0.1.3.1 (2026-06-20)
- Fixed duplicate (triple) game notifications: a games run with the Epic
  batch takes longer than the 5-minute scheduler interval, so overlapping
  runs polled games several times before the "checked" stamp was written.
  The `__last_games_check` stamp is now written and flushed BEFORE the run,
  not after — later runs correctly skip games until the next 24h window.

v0.1.3.0 (2026-06-19)
- Steam price comparison: the «Games» tab now has a "Epic now" column —
  the game's Epic Games Store price (click opens the EGS page). Fetched
  together with Steam via "Check now". No confident name match → "---";
  games flagged "🚫 don't check" are skipped on Epic too.
- When Epic is cheaper than Steam, the row turns gold, the Epic cell shows
  a 🔥, and a dedicated Telegram alert fires (its own antispam, independent
  of the Steam-sale alerts).
- The «Link» column on the Games tab became "Steam | Epic" — two clickable
  halves (the Epic half disappears when there's no Epic page).
- Separate games timer: "Check interval" renamed to "Cards… (min)", plus a
  new "Games check interval (h)" (default 24). Cards are polled every run,
  games no more often than this interval.
- Status-column text is now centered (all three lists).

v0.1.2.2 (2026-06-16)
- Game/card names with HTML entities (e.g. "Assassin&#x27;s") are now
  decoded to normal characters.
- Profile-background rarity (Common/Uncommon/Rare) is moved out of the game
  name into the type ("Uncommon Profile Background"); game name stays clean.

v0.1.2.1 (2026-06-15)
- watch.py no longer logs ERROR when config.json isn't there yet (fresh /
  clean install) — a calm log line instead of red.
- Installer also drops the legacy scheduled task (SteamCardWatch).
- Spare window icon (__app.ico) used if the primary one is missing/broken.

v0.1.2.0 (2026-06-15)
- Backup/restore: "Export" / "Import" buttons in Settings. Export saves
  config + all data into a .zip; import restores them (overwrite + restart).
  Format is interchangeable with the installer.

v0.1.1.0 (2026-06-15)
- "Do not disturb" mode: a time window (__:__ - __:__) during which
  Telegram alerts are not sent (even when antispam has expired). Checked
  in local time, works across midnight. Window start/end are logged.
